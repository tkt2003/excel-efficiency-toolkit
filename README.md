# Excel 效率工具台

基于 Python、Tkinter 和 pywin32 构建的轻量级 Excel 操作工具。

## 功能特性
- **安全低风险操作**：工具专注处理 Excel 本地文件和正在运行的 Excel 实例，不包含删除源文件、重命名源工作表及网络相关的高风险操作。
- **获取工作表列表**：一键获取当前活动工作簿中的所有工作表名称，并填入活动工作表的 A 列（从 A1 开始，A1为表头“工作表名称”）。
- **一簿按工作表拆分为多个文件**：选择一个源工作簿和输出目录后，按工作表分别导出为独立 `.xlsx` 文件。该功能需要 Windows + Microsoft Excel，源工作簿只读打开，不保存、不覆盖、不删除源文件；输出目录中已有同名文件时会自动追加 `_2`、`_3` 等序号。
- **多表合并到一个新表**：将当前活动工作簿中所有可见且有有效数据的工作表按行追加到一个新工作表，最左侧增加“来源工作表”列，不保存工作簿，不修改源工作表。
- **按指定列拆分当前表为多个工作表**：按当前活动工作表中指定列的不同值创建新工作表，空值归入“空白”工作表，不保存工作簿，不删除、重命名或覆盖源工作表；执行前必须先选中要拆分的源 sheet。
- **生成带链接的工作表目录**：在当前活动工作表 A 列写入工作表目录，并为每个工作表名称添加跳转到对应工作表 A1 的超链接；会覆盖当前活动 sheet 的 A 列目录区域，不保存工作簿。

## 技术栈
- 语言：Python 3
- GUI：Tkinter
- 核心库：pywin32 (win32com)

## 环境准备与依赖安装
本程序依赖 Windows 操作系统且需提前安装 Microsoft Excel。

```bash
pip install -r requirements.txt
```

## 运行方式
```bash
python -m src.excel_efficiency_toolkit.app
```

## 运行测试
本项目包含了不依赖真实 Excel 的单元测试（采用 Mock 机制）。
```bash
pytest tests/
```

## 使用前建议
请优先使用测试文件手工验收，不要直接对重要底稿操作。当前工作簿内的功能不会自动保存工作簿；确认结果正确后，再由用户自行决定是否保存。

## 手工验收步骤：获取工作表列表 (Windows + Excel)
1. 启动 Windows 下的 Microsoft Excel。
2. 新建或打开任意一个包含多个工作表的工作簿。
3. 确保目标工作簿是当前活跃的窗口。
4. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
5. 在界面上点击【列出当前工作簿所有工作表】按钮。
6. 查看 Excel 的当前活动工作表，A 列应该已被填入“工作表名称”以及对应的工作表列表。
7. **注意事项**：该操作不执行保存动作，也不会关闭您的 Excel 文件。

## 手工验收步骤：一簿按工作表拆分为多个文件 (Windows + Excel)
1. 准备一个包含多个工作表的 Excel 文件，例如 `源文件.xlsx`。
2. 准备一个空输出目录；也可以预先放入一个同名 `.xlsx` 文件，用于验证自动追加序号。
3. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
4. 点击【一簿按工作表拆分为多个文件】按钮。
5. 在文件选择框中选择源 Excel 文件。
6. 在文件夹选择框中选择输出目录。
7. 查看运行日志，应显示源文件、输出目录以及成功导出的文件数量。
8. 查看输出目录，每个工作表应各自生成一个 `.xlsx` 文件；如果已有同名文件，新文件名应自动追加 `_2`、`_3` 等序号。
9. 确认源工作簿未被保存、覆盖或删除。

## 手工验收步骤：多表合并到一个新表 (Windows + Excel)
1. 准备一个测试工作簿，创建多个可见工作表，每个工作表第 1 行放相同结构的表头，第 2 行开始放数据。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 确保测试工作簿是 Excel 当前活动工作簿。
4. 点击【多表合并到一个新表】按钮。
5. 依次输入结果 sheet 名、表头行号、数据起始行号；可使用默认值“合并结果”、`1`、`2`。
6. 查看 Excel 中新增的结果工作表，A 列应为“来源工作表”，后续列为第一个有效源工作表的表头。
7. 确认所有可见且有有效数据的工作表都已参与合并，日志会显示参与合并的 sheet 名、结果 sheet 名、合并 sheet 数和追加行数。
8. 不建议把目录页、说明页、辅助页混在测试工作簿中一起验收，避免它们也被识别为可合并工作表。
9. 确认源工作表未被删除、重命名、覆盖或修改，工作簿未被自动保存。

## 手工验收步骤：按指定列拆分当前表为多个工作表 (Windows + Excel)
1. 准备一个测试工作表，第 1 行为表头，第 2 行开始为数据，并确保某一列包含可分组的值。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 在 Excel 中必须先选中要拆分的源工作表，使其成为当前活动工作表。
4. 点击【按指定列拆分当前表为多个工作表】按钮。
5. 输入拆分列，例如 `A`、`B` 或 `1`、`2`，再输入表头行号和数据起始行号。
6. 查看 Excel 中新增的工作表，名称应来自拆分列的值；空值应归入“空白”工作表，重名时自动追加 `_2`、`_3`。
7. 确认每个新工作表包含原表头和对应数据行，日志显示源 sheet 名、生成 sheet 数和复制行数。
8. 确认源工作表未被删除、重命名、覆盖或修改，工作簿未被自动保存。

## 手工验收步骤：生成带链接的工作表目录 (Windows + Excel)
1. 准备一个包含多个工作表的测试工作簿。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 在 Excel 中选中用于放置目录的工作表，使其成为当前活动工作表。
4. 点击【生成带链接的工作表目录】按钮。
5. 查看当前活动工作表 A 列：A1 应为“工作表目录”，A2 起应列出所有工作表名称；该功能会覆盖当前活动 sheet 的 A 列目录区域。
6. 点击目录中的工作表名称，应跳转到对应工作表的 A1。
7. 确认日志显示工作表数量，工作簿未被自动保存。
