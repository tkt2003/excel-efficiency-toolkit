# Excel 效率工具台

基于 Python、Tkinter 和 pywin32 构建的轻量级 Excel 操作工具。

## 功能特性
- **安全低风险操作**：工具专注处理 Excel 本地文件和正在运行的 Excel 实例，不包含删除源文件及网络相关的高风险操作；涉及批量维护的功能会先生成规则表确认。
- **一簿按工作表拆分为多个文件**：选择一个源工作簿和输出目录后，按工作表分别导出为独立 `.xlsx` 文件。该功能需要 Windows + Microsoft Excel，源工作簿只读打开，不保存、不覆盖、不删除源文件；输出目录中已有同名文件时会自动追加 `_2`、`_3` 等序号。
- **多表合并到一个新表**：先选择源工作簿，再将该工作簿中所有可见且有有效数据的工作表按行追加到一个新工作表，最左侧增加“来源工作表”列；结果写入所选工作簿，不自动保存，不修改源工作表。
- **多簿到一簿**：在“合并整理”中点击【多簿到一簿】，多选源 `.xlsx` / `.xlsm` 文件，再选择一个已有目标 `.xlsx` / `.xlsm` 工作簿，将每个源文件的指定 sheet 导入目标工作簿。该功能全程使用 openpyxl，不使用 COM；源文件只读取、不保存、不修改，`.xls` 会跳过并提示另存为 `.xlsx` / `.xlsm`。目标工作簿会被修改并保存，执行前请关闭目标工作簿，程序会先在同目录生成备份。第一版支持复制值/公式、基础样式、行高列宽、合并单元格、冻结窗格、超链接、基础视图设置等；暂不承诺图片、图表、控件、数据透视表、宏模块、外部链接完整修复和复杂页面设置 100% 保真。
- **按指定列拆分工作表**：先选择源工作簿并输入源 sheet 名，再按指定列的不同值在同一工作簿中创建新工作表；空值归入“空白”工作表，不自动保存，不删除、重命名或覆盖源工作表。
- **生成带链接的工作表目录**：先选择源工作簿，自动新建或复用“工作表目录”sheet，并为其他工作表名称添加跳转到对应工作表 A1 的超链接；目录不包含目录 sheet 自己，不自动保存。
- **批量重命名工作表**：在“批量维护”中点击【批量重命名工作表】，多选 Excel 文件后生成临时规则表；在【重命名清单】D 列填写新工作表名。sheet 名不能超过 31 个字符，不能包含 `: \ / ? * [ ]`；目标名冲突默认自动编号，默认跳过隐藏工作表，支持链式改名和互换改名。执行后会修改并保存所选 Excel 文件，建议操作前先备份；状态和说明写入 E/F 列，并在【处理日志】记录明细。
- **批量更换多文件链接**：适用于一批已经生成好的 Excel 工作簿，需要批量扫描并替换这些工作簿中引用到的外部工作簿链接。在“批量维护”中点击【批量更换多文件链接】，多选目标工作簿后用 Excel COM 扫描 `Workbook.LinkSources`，自动生成临时规则表；在【链接清单】D 列填写新链接路径，E 列为空或“是”表示执行，“否”表示跳过。执行时按工作簿分组打开，默认先备份，再用 `Workbook.ChangeLink` 替换 Excel 外部工作簿链接并保存这些已有工作簿；状态写入 F/G 列，并在【处理日志】记录明细。该功能不会按模板批量生成新报表，仅在已有工作簿上原地替换链接；暂不处理 VBA、Power Query、数据透视表连接、ODBC/OLEDB 连接。
- **批量删除工作表**：选择源 Excel 文件后自动生成并打开临时规则表，用户填写并保存规则后回到工具台确认执行。该功能会直接修改并保存原 Excel 文件；A 列仅供参考，B 列用于保留模式，C 列用于删除模式，D 列用于排除文件名关键词。
- **批量重命名文件**：在“批量维护”中点击【批量重命名文件】，多选文件后会自动生成临时规则工作簿；【重命名清单】B 列展示原文件名主体，不含后缀，C 列填写新文件名，不含后缀，D 列为后缀名，默认原后缀，可修改。修改后缀名只改变文件名，不转换文件格式；目标文件已存在时默认自动编号，也可在【参数设置】改为跳过；程序不覆盖已有文件，会跳过非法文件名、原文件不存在等情况，执行后在 F/G 列写入状态和说明，并在【处理日志】记录明细。
- **按颜色汇总求和**：打开目标合并工作簿并选中一个带填充色的单元格后，可选择仅汇总一个 sheet，或汇总所有匹配 sheet。仅汇总一个 sheet 时可指定 sheet 名，或留空使用当前活动 sheet；汇总所有匹配 sheet 时会扫描目标工作簿所有可见 sheet，只处理包含相同填充色单元格的 sheet，隐藏 sheet 跳过。程序在多个源文件的同名 sheet 中按相同单元格地址逐格求和，不同 sheet 独立汇总、不互相混加，支持写入求和公式或只写入汇总数值。快速汇总使用 openpyxl 扫描目标模板颜色，目标工作簿必须是已保存的 `.xlsx` / `.xlsm`；如果刚修改了颜色、sheet 结构或汇总区域，需要先保存目标工作簿。目标颜色扫描不使用 COM 慢速兜底。源文件支持 `.xlsx` / `.xlsm`，只读读取、不保存、不修改；`.xls` 需另存为 `.xlsx` / `.xlsm` 后再汇总。公式单元格读取的是 Excel 上次保存的缓存计算结果。目标工作簿仍通过 Excel COM 写入且不自动保存；写入求和公式会生成外部链接，速度可能慢于只写入汇总数值，日常建议优先使用“只写入汇总数值”；写入结果会覆盖目标同色单元格原内容或公式。
- **数据穿透取数**：打开目标/合并工作簿后，进入要追查的 sheet 并选中单元格，再在“模板填报 / 附注汇总”中点击【数据穿透取数】。当前 Excel 只用于定位取数点和结果输出目录；程序会从多个源工作簿批量读取当前活动 sheet 同名 sheet、当前选中单元格同地址的值。源文件使用 openpyxl 只读读取，不保存、不修改；结果输出为当前活动工作簿同目录下独立的 `数据穿透结果.xlsx`，已有同名结果时自动编号。当前活动工作簿不会新增 sheet，不会自动保存。公式值依赖源文件最近一次保存后的缓存计算结果；`.xls` 需另存为 `.xlsx/.xlsm` 后再处理。
- **按清单生成报表**：在“模板填报 / 附注汇总”中点击【按清单生成报表】，选择模板工作簿、清单 sheet 和输出目录后，程序读取清单 sheet（默认 `tb文件名`，第 1 行为表头，A 列为新 TB / 新链接文件路径，B 列为输出文件名主体，C/D 列仅用于日志），逐行复制完整模板工作簿，再用 Excel COM 将模板外部链接全部替换为该行 A 列路径并保存。输出文件已存在时自动编号；原模板工作簿不会被修改。P0 不删除清单 sheet，不写入公司名称到模板单元格，不处理复杂多源映射。
- **按模板多选 TB 生成报表**：适用于“一个模板 + 多个 TB 文件”的场景，用户无需手工填写清单。在“模板填报 / 附注汇总”中点击【按模板多选 TB 生成报表】，选择模板工作簿（仅支持 `.xlsx` / `.xlsm`）后，程序用 Excel COM 只读扫描模板外部链接；若模板有多个外部链接，会弹出选择窗口让用户指定要替换的旧 TB 链接，只替换该链接，不替换其他外部链接。然后多选 TB 文件（仅支持 `.xlsx` / `.xlsm`，自动跳过 `~$` 临时文件和不支持格式）和输出目录，程序会对每个 TB 文件用 `shutil.copy2` 复制一份完整模板，输出文件名默认为 `TB 文件名主体_报表` 并保持模板后缀（模板为 `.xlsm` 时输出也为 `.xlsm`），目标文件已存在时自动编号、不覆盖；再用 Excel COM 打开复制出来的工作簿调用 `Workbook.ChangeLink` 替换为该 TB 路径并保存。原模板工作簿不会被修改、保存或覆盖；单个 TB 失败不会中断整批；输出目录会同时生成处理日志 `按模板多选TB生成报表_处理日志.xlsx`，记录每个 TB 文件的状态、说明、被替换的旧链接和新链接。
- **选区 ROUND 保留两位**：打开 Excel 工作簿并选中区域后，在“模板填报 / 附注汇总”中点击【选区 ROUND 保留两位】，程序会直接把选区内数值常量改为 `=ROUND(原数值,2)`，把普通公式或外部链接公式改为 `=ROUND(原公式去掉等号,2)`；空白、文本、错误值和已经是 `ROUND(...)` 的公式会跳过。该功能只修改当前选区，不自动保存当前工作簿，不生成备份。

## 技术栈
- 语言：Python 3
- GUI：Tkinter
- 核心库：pywin32 (win32com)、openpyxl

## 环境准备与依赖安装
本程序依赖 Windows 操作系统且需提前安装 Microsoft Excel。

```bash
pip install -r requirements.txt
```

## 运行方式
```bash
python run_app.py
```

也可以使用模块方式运行：

```bash
python -m src.excel_efficiency_toolkit.app
```

## 打包 Windows exe
项目提供 PyInstaller 打包脚本，默认入口为 `run_app.py`，输出到 `dist/Excel效率工具台.exe`。

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_exe.ps1
```

脚本会先检查 PyInstaller；如未安装会尝试通过 `python -m pip install pyinstaller` 安装。`build/`、`dist/` 和自动生成的 `.spec` 文件默认不纳入版本控制。

## 运行测试
本项目包含了不依赖真实 Excel 的单元测试（采用 Mock 机制）。
```bash
pytest tests/
```

## 使用前建议
请优先使用测试文件手工验收，不要直接对重要底稿操作。批量删除工作表会直接修改并保存原 Excel 文件，使用前请自行备份。

## 手工验收步骤：批量更换多文件链接 (Windows + Excel)
1. 准备 `old_tb.xlsx`、`new_tb.xlsx` 和一个公式引用 `old_tb.xlsx` 的 `target.xlsx`。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 在“批量维护”中点击【批量更换多文件链接】，选择 `target.xlsx`。
4. 检查生成的规则表包含“使用说明”“参数设置”“链接清单”“处理日志”。
5. 在【链接清单】D 列填写 `new_tb.xlsx` 的完整路径，保存并关闭规则表。
6. 回到工具台点击【我已填好规则，执行批量更换多文件链接】。
7. 检查 `target.xlsx` 同目录已生成备份，目标工作簿外部链接已替换为 `new_tb.xlsx`。
8. 检查规则表 F/G 列状态为成功，【处理日志】有对应明细。

## 手工验收步骤：按清单生成报表 (Windows + Excel)
1. 准备模板工作簿，包含 `tb文件名` sheet，且模板内有公式引用 `old_tb.xlsx`。
2. 在 `tb文件名` sheet 第 1 行放表头，第 2 行起填写：A 列为 `new_tb_1.xlsx` / `new_tb_2.xlsx` 完整路径，B 列为 `子公司1` / `子公司2`。
3. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
4. 在“模板填报 / 附注汇总”中点击【按清单生成报表】。
5. 选择模板工作簿，清单 sheet 使用默认 `tb文件名`，再选择输出目录。
6. 确认后执行，检查输出目录生成 `子公司1.xlsx` 和 `子公司2.xlsx`。
7. 检查两个输出文件的外部链接分别替换为对应的 A 列 TB 文件。
8. 检查原模板工作簿未被修改。

## 手工验收步骤：按模板多选 TB 生成报表 (Windows + Excel)
1. 准备 `old_tb.xlsx`，`Sheet1!A1 = 111`；`keep_link.xlsx`，`Sheet1!A1 = 999`；`tb_子公司1.xlsx`，`Sheet1!A1 = 1001`；`tb_子公司2.xlsx`，`Sheet1!A1 = 2002`。
2. 准备 `template.xlsx`，其中一个单元格公式引用 `old_tb.xlsx`，另一个单元格公式引用 `keep_link.xlsx`。这样模板有 2 个外部链接。
3. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
4. 在“模板填报 / 附注汇总”中点击【按模板多选 TB 生成报表】。
5. 选择 `template.xlsx`。
6. 在外部链接选择弹窗中，选择 `old_tb.xlsx` 对应的链接作为要替换的旧链接。
7. 多选 `tb_子公司1.xlsx` 和 `tb_子公司2.xlsx`。
8. 选择输出目录。
9. 检查输出目录生成 `tb_子公司1_报表.xlsx` 和 `tb_子公司2_报表.xlsx`。
10. 检查 `tb_子公司1_报表.xlsx`：原指向 `old_tb.xlsx` 的链接已变为 `tb_子公司1.xlsx`；原指向 `keep_link.xlsx` 的链接保持不变。
11. 检查 `tb_子公司2_报表.xlsx`：原指向 `old_tb.xlsx` 的链接已变为 `tb_子公司2.xlsx`；原指向 `keep_link.xlsx` 的链接保持不变。
12. 检查 `template.xlsx` 仍引用 `old_tb.xlsx` 和 `keep_link.xlsx`，模板未被修改。
13. 检查输出目录生成 `按模板多选TB生成报表_处理日志.xlsx`，包含 2 条成功记录。

## 手工验收步骤：选区 ROUND 保留两位 (Windows + Excel)
1. 打开一个临时工作簿。
2. A1 输入 `123.456`，A2 输入 `=1/3`，A3 输入文本。
3. 选中 A1:A3。
4. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
5. 在“模板填报 / 附注汇总”中点击【选区 ROUND 保留两位】。
6. 检查 A1 变成 `=ROUND(123.456,2)`，A2 变成 `=ROUND(1/3,2)`，A3 保持文本不变。
7. 检查当前工作簿未自动保存，请确认后自行保存或放弃。

## 手工验收步骤：按颜色汇总求和 (Windows + Excel)
1. 准备一个目标合并工作簿，sheet 名为“货币资金”，C4/C5/D4 等单元格填黄色。
2. 准备 2 个源工作簿，也包含“货币资金”sheet，并在相同地址填入数字。
3. 打开目标合并工作簿，选中一个带黄色填充的单元格；如果刚修改了颜色或汇总区域，先保存目标工作簿。
4. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
5. 点击【按颜色汇总求和】按钮。
6. 输入“货币资金”，测试指定 sheet 名。
7. 多选源工作簿。
8. 选择“只写入汇总数值”。
9. 检查目标 C4 = 两个源文件 C4 之和，C5 = 两个源文件 C5 之和，不是把所有黄色单元格混合求和。
10. 再测试“写入求和公式”模式。
11. 再次点击【按颜色汇总求和】，sheet 名输入框留空，测试使用当前活动 sheet。
12. 确认 `.xls` 源文件会提示需另存为 `.xlsx` / `.xlsm`，源文件未被修改，目标工作簿未自动保存。

## 手工验收步骤：按颜色汇总求和 - 所有匹配 sheet (Windows + Excel)
1. 准备一个目标合并工作簿，至少包含“货币资金”“交易性金融资产”两个可见 sheet，并在 C4/C5 等需要汇总的单元格填相同黄色。
2. 准备 2 个源工作簿，也包含同名 sheet，并在相同地址填入不同数字。
3. 打开目标合并工作簿，选中一个黄色单元格；如果刚修改了颜色、sheet 结构或汇总区域，先保存目标工作簿。
4. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
5. 点击【按颜色汇总求和】按钮。
6. 选择“汇总所有匹配 sheet”。
7. 多选源工作簿。
8. 选择“只写入汇总数值”。
9. 检查“货币资金”!C4 = 各源文件“货币资金”!C4 之和，“交易性金融资产”!C4 = 各源文件“交易性金融资产”!C4 之和，两个 sheet 不互相混加。
10. 再测试“写入求和公式”模式。
11. 确认隐藏 sheet 被跳过，`.xls` 源文件会提示需另存为 `.xlsx` / `.xlsm`，源文件未被修改，目标工作簿未自动保存。

## 手工验收步骤：数据穿透取数 (Windows + Excel)
1. 准备一个目标/合并工作簿并保存到测试目录，sheet 名为“货币资金”。
2. 打开该目标工作簿，进入“货币资金”sheet，选中 C5。
3. 准备 3 个源文件：源1 有“货币资金”!C5 普通数值；源2 有“货币资金”!C5 公式并保存；源3 没有“货币资金”sheet，用于验证跳过。
4. 可选准备 `~$临时.xlsx` 和 `.xls` 文件，用于验证跳过。
5. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
6. 点击【数据穿透取数】。
7. 确认弹窗显示取数点为“货币资金!C5”，并提示结果生成到当前工作簿同目录、当前工作簿不会被修改或保存。
8. 多选源文件。
9. 检查当前工作簿同目录生成 `数据穿透结果.xlsx`。
10. 检查结果文件字段为“序号、源文件名、源文件路径、目标 Sheet 名、单元格地址、取值、是否为空、是否错误值、状态、说明”。
11. 检查结果文件中记录普通值、公式缓存值、空值、错误值、缺少 sheet 状态。
12. 检查源文件未被修改，当前目标工作簿没有新增 sheet、没有被自动保存。
13. 再次执行，确认生成 `数据穿透结果_2.xlsx`，不覆盖已有结果文件。

## 手工验收步骤：一簿按工作表拆分为多个文件 (Windows + Excel)
1. 准备一个包含多个工作表的 Excel 文件，例如 `源文件.xlsx`。
2. 准备一个空输出目录；也可以预先放入一个同名 `.xlsx` 文件，用于验证自动追加序号。
3. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
4. 点击【一簿按工作表拆分为多个文件】按钮。
5. 在文件选择框中选择源 Excel 文件。
6. 在文件夹选择框中选择输出目录。
7. 查看运行日志，应显示源文件、输出目录以及成功导出的文件数量。
8. 查看输出目录，每个工作表应各自生成一个 `.xlsx` 文件；如果已有同名文件，新文件名应自动追加 `_2`、`_3` 等序号。
9. 确认源工作簿未被保存、覆盖或删除。

## 手工验收步骤：多表合并到一个新表 (Windows + Excel)

9. 确认处理结果写入所选工作簿，源工作表未被删除、重命名、覆盖或修改，工作簿未被自动保存；检查无误后由用户自行保存。

11. 再跑一次“是，只复制值”，验证公式缓存为空时保留公式，并在日志说明“公式缓存为空，已保留公式”。

## 手工验收步骤：批量重命名工作表 (Windows + Excel)
1. 准备 2-3 个已备份的临时测试工作簿，包含多个测试 sheet。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 在“批量维护”中点击【批量重命名工作表】。
4. 在文件选择框中多选测试 Excel 文件。
5. 程序会读取所选工作簿的 sheet 清单，并生成包含“使用说明”“参数设置”“重命名清单”“处理日志”的临时规则表。
6. 在“重命名清单”D 列填写新工作表名；名称不能超过 31 个字符，不能包含 `: \ / ? * [ ]`。
7. 保存并关闭规则工作簿。
8. 回到工具台弹窗，点击【我已填好规则，执行批量重命名工作表】。
9. 检查所选工作簿中的 sheet 已按规则重命名并保存；目标名冲突时默认追加 `_1`、`_2`，隐藏工作表默认跳过。
10. 可测试 `Sheet1 -> Sheet2 -> Sheet3` 和 `A <-> B`，确认不会生成错误编号。
11. 检查规则工作簿 E/F 列已写入状态和说明，“处理日志”sheet 已记录明细。

## 手工验收步骤：批量删除工作表 (Windows + Excel)
1. 准备 2-3 个已备份的测试 Excel 文件，包含不同表格名，可以故意加入空格、特殊字符。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 点击【批量删除工作表】按钮。
4. 在文件选择框中多选这些测试 Excel 文件。
5. 程序会自动生成并打开临时规则表；检查“批量删除规则”sheet 的 A1:D1 表头应为“所有表格名”“保留表格名”“删除表格名”“排除文件名关键词”。
6. 检查 A 列列出所有出现过的表格名，重复表格名只出现一次，且首次出现顺序保留。
7. 在规则表中填写规则：B 列填写要保留的表格名，或 C 列填写要删除的表格名，D 列可填写某个文件名关键词用于跳过文件；A 列仅供参考，不参与执行。
8. 保存规则表。
9. 回到工具台弹窗，点击【我已填好规则，执行批量删除】。
10. 如果只有 B 列有内容，程序自动使用保留模式；如果只有 C 列有内容，程序自动使用删除模式；如果 B/C 都有内容，程序会询问执行模式；如果 B/C 都为空，程序会报错。
11. 检查原 Excel 文件已按规则直接删除工作表并保存。
12. 检查被 D 列关键词命中的文件被跳过。
13. 检查不会生成 0 sheet 工作簿；如果规则会删空某个工作簿，该文件应失败并记录日志。

## 手工验收步骤：批量重命名文件 (Windows + Excel)
1. 准备一个临时文件夹，放入 2-3 个测试文件，不要使用真实业务文件。

6. 在“重命名清单”中检查 B 列原文件名主体；C 列填写新文件名，不含后缀；D 列可保留原后缀，也可改为 `.xlsx` 或 `xlsx` 这类格式。
7. 保存并关闭规则工作簿。
8. 回到工具台弹窗，点击【我已填好规则，执行批量重命名】。

10. 检查规则工作簿 F/G 列已写入状态和说明，“处理日志”sheet 已记录成功、跳过或失败明细。
