# 老头表格助手

基于 Python、Tkinter 和 pywin32 构建的轻量级 Excel 操作工具。

## 功能特性
- **安全低风险操作**：工具专注处理 Excel 本地文件和正在运行的 Excel 实例，不包含删除源文件及网络相关的高风险操作；涉及批量维护的功能会先生成规则表确认。
- **一簿按工作表拆分为多个文件**：选择一个源工作簿和输出目录后，按工作表分别导出为独立 `.xlsx` 文件。该功能需要 Windows + Microsoft Excel，源工作簿只读打开，不保存、不覆盖、不删除源文件；输出目录中已有同名文件时会自动追加 `_2`、`_3` 等序号。
- **多表合并到一个新表**：先选择源工作簿，再将该工作簿中所有可见且有有效数据的工作表按行追加到一个新工作表，最左侧增加“来源工作表”列；结果写入所选工作簿，不自动保存，不修改源工作表。
- **多簿到一簿**：在“合并整理”中点击【多簿到一簿】，多选源 `.xlsx` / `.xlsm` 文件，再选择一个已有目标 `.xlsx` / `.xlsm` 工作簿，将每个源文件的指定 sheet 导入目标工作簿。该功能全程使用 openpyxl，不使用 COM；源文件只读取、不保存、不修改，`.xls` 会跳过并提示另存为 `.xlsx` / `.xlsm`。目标工作簿会被修改并保存，执行前请关闭目标工作簿，程序会先在同目录生成备份。第一版支持复制值/公式、基础样式、行高列宽、合并单元格、冻结窗格、超链接、基础视图设置等；暂不承诺图片、图表、控件、数据透视表、宏模块、外部链接完整修复和复杂页面设置 100% 保真。
- **按指定列拆分工作表**：先选择源工作簿并输入源 sheet 名，再按指定列的不同值在同一工作簿中创建新工作表；空值归入“空白”工作表，不自动保存，不删除、重命名或覆盖源工作表。
- **生成带链接的工作表目录**：先选择源工作簿，自动新建或复用“工作表目录”sheet，并为其他工作表名称添加跳转到对应工作表 A1 的超链接；目录不包含目录 sheet 自己，不自动保存。
- **批量重命名工作表**：在“批量维护”中点击【批量重命名工作表】，多选 Excel 文件后生成临时规则表；在【重命名清单】D 列填写新工作表名。sheet 名不能超过 31 个字符，不能包含 `: \ / ? * [ ]`；目标名冲突默认自动编号，默认跳过隐藏工作表，支持链式改名和互换改名。执行后会修改并保存所选 Excel 文件，建议操作前先备份；状态和说明写入 E/F 列，并在【处理日志】记录明细。
- **批量更换多文件链接**：适用于一批已经生成好的 Excel 工作簿，需要批量扫描并替换这些工作簿中引用到的外部工作簿链接。在“批量维护”中点击【批量更换多文件链接】，多选目标工作簿后用 Excel COM 扫描 `Workbook.LinkSources`，自动生成临时规则表；在【链接清单】D 列填写新链接路径，E 列为空或“是”表示执行，“否”表示跳过。执行时按工作簿分组打开，默认先备份，再用 `Workbook.ChangeLink` 替换 Excel 外部工作簿链接并保存这些已有工作簿；状态写入 F/G 列，并在【处理日志】记录明细。该功能不会按模板批量生成新 Excel 文件，仅在已有工作簿上原地替换链接；暂不处理 VBA、Power Query、数据透视表连接、ODBC/OLEDB 连接。
- **批量删除工作表**：选择源 Excel 文件后自动生成并打开临时规则表，用户填写并保存规则后回到工具台确认执行。该功能会直接修改并保存原 Excel 文件；A 列仅供参考，B 列用于保留模式，C 列用于删除模式，D 列用于排除文件名关键词。
- **批量重命名文件**：在“批量维护”中点击【批量重命名文件】，多选文件后会自动生成临时规则工作簿；【重命名清单】B 列展示原文件名主体，不含后缀，C 列填写新文件名，D 列为后缀名，默认原后缀，可修改。新文件名不填写后缀时，程序默认拼接规则表当前“后缀名”列；如果新文件名已显式填写后缀，则以“新文件名”列完整内容为准，不再拼接 D 列；只有 D 列为空时，才回退使用原文件真实后缀。修改后缀名只改变文件名，不转换文件格式；目标文件已存在时默认自动编号，也可在【参数设置】改为跳过；程序不覆盖已有文件，会跳过非法文件名、原文件不存在等情况，执行后在 F/G 列写入状态和说明，并在【处理日志】记录明细。
- **按颜色汇总求和**：打开目标合并工作簿并选中一个带填充色的单元格后，可选择仅汇总一个 sheet，或汇总所有匹配 sheet。仅汇总一个 sheet 时可指定 sheet 名，或留空使用当前活动 sheet；汇总所有匹配 sheet 时会扫描目标工作簿所有可见 sheet，只处理包含相同填充色单元格的 sheet，隐藏 sheet 跳过。程序在多个源文件的同名 sheet 中按相同单元格地址逐格求和，不同 sheet 独立汇总、不互相混加，支持写入求和公式或只写入汇总数值。快速汇总使用 openpyxl 扫描目标模板颜色，目标工作簿必须是已保存的 `.xlsx` / `.xlsm`；如果刚修改了颜色、sheet 结构或汇总区域，需要先保存目标工作簿。目标颜色扫描不使用 COM 慢速兜底。源文件支持 `.xlsx` / `.xlsm`，只读读取、不保存、不修改；`.xls` 需另存为 `.xlsx` / `.xlsm` 后再汇总。公式单元格读取的是 Excel 上次保存的缓存计算结果。目标工作簿仍通过 Excel COM 写入且不自动保存；写入求和公式会生成外部链接，速度可能慢于只写入汇总数值，日常建议优先使用“只写入汇总数值”；写入结果会覆盖目标同色单元格原内容或公式。
- **按颜色清空内容**：在“模板生成 / 取数”中点击【按颜色清空内容】，先读取当前活动单元格的填充颜色，再选择“当前活动工作簿”或“多个工作簿”。当前活动工作簿模式只用 COM 读取当前工作簿、工作表、左上角选区、填充颜色、工作簿路径和是否存在未保存修改；要求当前工作簿已保存且没有未保存修改，然后关闭当前工作簿、生成备份，再用 openpyxl 扫描并清空所有可见 sheet 中同颜色单元格的值和公式，保存原文件，最后重新打开供用户检查。多个工作簿模式会对所选 `.xlsx` / `.xlsm` / `.xltx` / `.xltm` 文件用 openpyxl 扫描所有可见 sheet 的同色单元格，先备份，再清空值和公式并保存；格式、填充色、字体、边框、批注和合并状态都会保留；`.xls` 暂不支持，会跳过并记入处理日志。
- **数据穿透查询**：在“模板生成 / 取数”中点击【数据穿透查询】后，可选择“单文件数据穿透查询”或“多文件数据穿透查询”，两种模式都支持单个单元格（如 `C5`）和单一连续区域（如 `B2:D8`），不支持 Ctrl 多选的多区域选区。单文件模式会读取当前活动工作簿中所有可见 sheet 的同一地址单元格或区域，并直接在当前工作簿新增 `数据穿透查询结果` sheet，已有同名结果 sheet 会自动编号，隐藏 sheet 和历史结果 sheet 会跳过，当前工作簿不会自动保存。多文件模式会从多个源工作簿读取当前活动 sheet 同名 sheet 的同一地址单元格或区域，输出独立的 `数据穿透查询结果.xlsx`，已有同名结果文件时自动编号；源文件和当前活动工作簿都不会被修改或自动保存。公式值依赖 Excel 最近一次保存后的缓存计算结果；`.xls` 需另存为 `.xlsx/.xlsm` 后再处理。
- **按模板批量生成 Excel（统一入口）**：在“模板生成 / 取数”中点击【按模板批量生成 Excel】，当前主入口弹窗只提供两个常用模式：`替换一个链接生成` 适合模板里主要更换一个 TB / 附注 / 底稿链接；`替换多个链接生成` 适合模板里有多个外部链接，需要通过规则表逐项替换。清单驱动版逻辑仍保留，但当前首页入口暂不展示。
- **按模板批量生成 Excel · 清单驱动版（当前首页入口暂不展示）**：清单驱动版逻辑仍保留，适用于已有 `tb文件名` 清单 sheet 的场景。选择模板工作簿、清单 sheet 和输出目录后，程序读取清单 sheet（默认 `tb文件名`，第 1 行为表头，A 列为新 TB / 新链接文件路径，B 列为输出文件名主体，C/D 列仅用于日志），逐行复制完整模板工作簿，再用 Excel COM 将模板外部链接全部替换为该行 A 列路径并保存。输出文件已存在时自动编号；原模板工作簿不会被修改。P0 不删除清单 sheet，不写入公司名称到模板单元格，不处理复杂多源映射。
- **按模板批量生成 Excel · 替换一个链接生成**：单链接快速版，适用于“一个模板 + 多个 TB 文件”的场景，用户无需手工填写清单，可按模板批量生成 Excel 文件，不限于报表，也可用于附注、底稿等。在统一入口弹窗中选择【替换一个链接生成】，选择模板工作簿（仅支持 `.xlsx` / `.xlsm`）后，程序用 Excel COM 只读扫描模板外部链接；若模板有多个外部链接，会弹出选择窗口让用户指定要替换的旧 TB 链接，只替换该链接，不替换其他外部链接。然后多选 TB 文件（仅支持 `.xlsx` / `.xlsm`，自动跳过 `~$` 临时文件和不支持格式）和输出目录，程序会对每个 TB 文件用 `shutil.copy2` 复制一份完整模板。输出文件名默认使用 `TB 文件名主体_批量生成 + 模板后缀`；用户可以在执行流程中修改命名后缀，例如 `_附注`、`_财务报表`、`_底稿`、`_报表`；留空命名后缀时，输出文件名直接使用 `TB 文件名主体 + 模板后缀`；模板是 `.xlsm` 时输出仍为 `.xlsm`；输出文件已存在时自动编号、不覆盖。再用 Excel COM 打开复制出来的工作簿调用 `Workbook.ChangeLink` 替换为该 TB 路径并保存。原模板工作簿不会被修改、保存或覆盖；单个 TB 失败不会中断整批；输出目录会同时生成处理日志 `按模板批量生成Excel_处理日志.xlsx`，记录每个 TB 文件的状态、说明、被替换的旧链接和新链接。
- **按模板批量生成 Excel · 替换多个链接生成**：多链接规则表版，适用于模板中有合并 TB、单体 TB 等多个外部链接，且不是每个输出文件都需要替换所有链接、不同链接源文件数量不同或顺序不完全一致的场景。在统一入口弹窗中选择【替换多个链接生成】，选择模板后先指定主链接并多选主链接的新源文件；主链接决定生成清单行数，程序在输出目录生成规则表并自动填写主链接的新链接列。其他旧链接的新链接列默认留空，留空表示不替换，用户执行前可在规则表中人工填写或校正。确认执行后，程序逐行复制模板并对该行填写的新链接逐个调用 `Workbook.ChangeLink`；原模板不修改，输出文件已存在时自动编号、不覆盖，规则表会回写状态和处理日志。
- **选区 ROUND 保留两位**：打开 Excel 工作簿并选中区域后，在“模板生成 / 取数”中点击【选区 ROUND 保留两位】，程序会直接把选区内数值常量改为 `=ROUND(原数值,2)`，把普通公式或外部链接公式改为 `=ROUND(原公式去掉等号,2)`；空白、文本、错误值和已经是 `ROUND(...)` 的公式会跳过。该功能只修改当前选区，不自动保存当前工作簿，不生成备份。

## 技术栈
- 语言：Python 3
- GUI：Tkinter
- 核心库：pywin32 (win32com)、openpyxl

## 环境准备与依赖安装
本程序依赖 Windows 操作系统且需提前安装 Microsoft Excel。

```bash
pip install -r requirements.txt
```

## 运行方式
```bash
python run_app.py
```

也可以使用模块方式运行：

```bash
python -m src.excel_efficiency_toolkit.app
```

## 打包 Windows exe
项目提供 PyInstaller 打包脚本，默认入口为 `run_app.py`，输出到 `dist/老头表格助手.exe`。

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_exe.ps1
```

脚本会先检查 PyInstaller；如未安装会尝试通过 `python -m pip install pyinstaller` 安装。`build/`、`dist/` 和自动生成的 `.spec` 文件默认不纳入版本控制。

## 运行测试
本项目包含了不依赖真实 Excel 的单元测试（采用 Mock 机制）。
```bash
pytest tests/
```

## 使用前建议
请优先使用测试文件手工验收，不要直接对重要底稿操作。批量删除工作表会直接修改并保存原 Excel 文件，使用前请自行备份。

## 手工验收步骤：批量更换多文件链接 (Windows + Excel)
1. 准备 `old_tb.xlsx`、`new_tb.xlsx` 和一个公式引用 `old_tb.xlsx` 的 `target.xlsx`。
2. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
3. 在“批量维护”中点击【批量更换多文件链接】，选择 `target.xlsx`。
4. 检查生成的规则表包含“使用说明”“参数设置”“链接清单”“处理日志”。
5. 在【链接清单】D 列填写 `new_tb.xlsx` 的完整路径，保存并关闭规则表。
6. 回到工具台点击【我已填好规则，执行批量更换多文件链接】。
7. 检查 `target.xlsx` 同目录已生成备份，目标工作簿外部链接已替换为 `new_tb.xlsx`。
8. 检查规则表 F/G 列状态为成功，【处理日志】有对应明细。

## 手工验收步骤：按模板批量生成 Excel · 替换一个链接生成 (Windows + Excel)
1. 准备 `old_tb.xlsx`，`Sheet1!A1 = 111`；`keep_link.xlsx`，`Sheet1!A1 = 999`；`tb_子公司1.xlsx`，`Sheet1!A1 = 1001`；`tb_子公司2.xlsx`，`Sheet1!A1 = 2002`。
2. 准备 `template.xlsx`，其中一个单元格公式引用 `old_tb.xlsx`，另一个单元格公式引用 `keep_link.xlsx`。这样模板有 2 个外部链接。
3. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
4. 在“模板生成 / 取数”中点击【按模板批量生成 Excel】，在弹窗中选择【替换一个链接生成】。
5. 选择 `template.xlsx`。
6. 在外部链接选择弹窗中，选择 `old_tb.xlsx` 对应的链接作为要替换的旧链接。
7. 多选 `tb_子公司1.xlsx` 和 `tb_子公司2.xlsx`。
8. 选择输出目录。
9. 命名后缀保持默认 `_批量生成`；也可改成 `_附注`、`_财务报表`、`_底稿`、`_报表`，或留空直接使用 TB 文件名主体。
10. 检查输出目录生成 `tb_子公司1_批量生成.xlsx` 和 `tb_子公司2_批量生成.xlsx`。
11. 检查 `tb_子公司1_批量生成.xlsx`：原指向 `old_tb.xlsx` 的链接已变为 `tb_子公司1.xlsx`；原指向 `keep_link.xlsx` 的链接保持不变。
12. 检查 `tb_子公司2_批量生成.xlsx`：原指向 `old_tb.xlsx` 的链接已变为 `tb_子公司2.xlsx`；原指向 `keep_link.xlsx` 的链接保持不变。
13. 检查 `template.xlsx` 仍引用 `old_tb.xlsx` 和 `keep_link.xlsx`，模板未被修改。
14. 检查输出目录生成 `按模板批量生成Excel_处理日志.xlsx`，包含 2 条成功记录。

## 手工验收步骤：按模板批量生成 Excel · 替换多个链接生成 (Windows + Excel)
1. 准备 `old_single.xlsx`、`old_group.xlsx`、`single_公司1.xlsx`、`single_公司2.xlsx`、`group_公司1.xlsx`。
2. 准备 `template.xlsx`，其中一个公式引用 `old_single.xlsx`，另一个公式引用 `old_group.xlsx`。
3. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
4. 在“模板生成 / 取数”中点击【按模板批量生成 Excel】，在弹窗中选择【替换多个链接生成】。
5. 选择 `template.xlsx`，主链接选择 `old_single.xlsx`，主源文件多选 `single_公司1.xlsx` 和 `single_公司2.xlsx`。
6. 输出后缀保持默认 `_批量生成`，选择输出目录后检查自动打开的规则表。
7. 在【生成清单】中，第 1 行为 `old_group.xlsx` 的新链接列填写 `group_公司1.xlsx`，第 2 行留空；空新链接表示不替换。
8. 保存并关闭规则表，回到工具台点击【我已检查规则，开始生成】。
9. 检查第 1 个输出文件同时替换 `old_single.xlsx` 和 `old_group.xlsx`，第 2 个输出文件只替换 `old_single.xlsx`，原模板未被修改。
10. 检查规则表【生成清单】状态和【处理日志】已回写成功记录。

## 手工验收步骤：选区 ROUND 保留两位 (Windows + Excel)
1. 打开一个临时工作簿。
2. A1 输入 `123.456`，A2 输入 `=1/3`，A3 输入文本。
3. 选中 A1:A3。
4. 启动本工具台：`python -m src.excel_efficiency_toolkit.app`。
5. 在“模板生成 / 取数”中点击【选区 ROUND 保留两位】。
6. 检查 A1 变成 `=ROUND(123.456,2)`，A2 变成 `=ROUND(1/3,2)`，A3 保持文本不变。
7. 检查当前工作簿未自动保存，请确认后自行保存或放弃。

## 手工验收步骤：按颜色汇总求和 (Windows + Excel)
1. 准备一个目标合并工作簿，sheet 名为“货币资金”，C4/C5/D4 等单元格填黄色。
2. 准备 2 个源工作簿，也包含“货币资金”sheet，并在相同地址填入数字。
3. 打开目标合并工作簿，选中一个带黄色填充的单元格；如果刚修改了颜色或汇总区域，先保存目标工作簿。
4. 运行本工具台：`python -m src.excel_efficiency_toolkit.app`。
